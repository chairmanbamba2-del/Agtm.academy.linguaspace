Système de Permissions IA

Hiérarchie des Permissions :

  1. Permissions utilisateur spécifiques (priorité haute)
  2. Permissions par plan (UNI / All Access)
  3. Paramètres globaux (configuration système)
  4. Configuration statique (fallback dans ai-config.js)

Fonctionnalités du système :

  • Limites de tokens quotidiennes configurables
  • Modèles par défaut par type de session (free_talk, business, etc.)
  • Activation/désactivation de la recherche web
  • Surveillance de l'utilisation (tokens, coûts)
  • Interface admin complète pour la gestion
